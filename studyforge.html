StudyForge

Forge any lecture into a study habit.

StudyForge turns pasted notes, PDFs, or YouTube lectures into spaced-repetition flashcards using AI — then schedules every review with the SM-2 algorithm and grades your free-text answers in real time.

React + Vite Express 5 PostgreSQL + Drizzle OpenAPI codegen OpenAI SM-2 scheduling

Features

AI deck generation

Paste text, drop a PDF, or hand it a YouTube URL. The model writes a deck summary plus a balanced mix of cards.

Difficulty-tagged cards

Recall Conceptual Applied Roughly 50 / 30 / 20 split so you build fluency, not just memory.

Free-text answer grading

Type your answer in your own words. The AI returns a verdict, a 0–5 score, and an adaptive follow-up question on partial answers.

SM-2 spaced repetition

Each rating (again / hard / good / easy) updates the card's ease factor and schedules the next review for exactly when you'd start to forget.

Dashboard & streaks

See cards due today, current streak, mastery distribution (new → learning → young → mature), and a 30-day activity heatmap.

Mobile-friendly study UI

Sidebar on desktop, bottom nav on mobile. Polished transitions powered by Framer Motion.

How it works

  1. Ingest: Pasted text is sent as-is, PDFs are parsed in the browser with pdfjs-dist, and YouTube URLs are converted to transcripts on the server.
  2. Generate: The source is sent to GPT with a strict JSON schema. It returns a deck summary and an array of cards tagged by difficulty.
  3. Study: When a card is due, you type a free-text answer. A second AI call grades it and suggests a SM-2 rating.
  4. Schedule: The SM-2 algorithm updates easeFactor, intervalDays, and repetitions, then sets the next dueAt.
  5. Review: The dashboard tracks streaks, mastery, and your daily review activity.

Getting started

Clone the repo and install dependencies:

git clone https://github.com/your-username/studyforge.git
cd studyforge
pnpm install

Set the required environment variables (a .env file at the repo root works):

DATABASE_URL=postgres://user:password@host:5432/studyforge
AI_INTEGRATIONS_OPENAI_BASE_URL=https://api.openai.com/v1
AI_INTEGRATIONS_OPENAI_API_KEY=sk-...
SESSION_SECRET=replace-me

Push the database schema and start the workflows:

pnpm --filter @workspace/db run db:push
pnpm --filter @workspace/api-server run dev   # backend on :8080
pnpm --filter @workspace/studyforge run dev   # frontend on :19683

Open the app at http://localhost (the dev proxy routes /api to the server and everything else to the web client).
